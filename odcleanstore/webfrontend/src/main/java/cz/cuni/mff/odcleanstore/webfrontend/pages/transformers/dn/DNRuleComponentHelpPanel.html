DN rule component

Every component is of INSERT / DELETE / MODIFY type. This affects the behaviour of the component and its definition.

INSERTQuadPattern WHERE GroupGraphPattern
example:{?s ?p ?o} WHERE {GRAPH $$graph$$ {SELECT ?s ?p 'Y' AS ?o WHERE {?s ?p 1}}}
DELETEQuadPattern WHERE GroupGraphPattern
example:{?s ?p 1} WHERE {GRAPH $$graph$$ {?s ?p 1}}
MODIFYDELETE QuadPattern INSERT QuadPattern WHERE GroupGraphPattern
example:DELETE {?s ?p 1} INSERT {?s ?p 'Y'} WHERE {?s ?p 1}

The examples above are intentionally overcomplicated to capture different aspects of the modifications

Subqueries are used to bind variables instead of SPARQL 1.1 BIND for compatibility reasons. This is also the reason for introduction of $$graph$$ which is a placeholder for currently processed graph and should be used only in GraphGraphPattern.
