Graphs in Error

Each graph processed by the selected pipeline can be processed by the pipeline once again with the button (this transformation will reflect current state of the pipeline configuration) or it can be deleted by button. Graphs that are in clean database can in addition be accepted () as they are despite the errors if the user considers them irrelevant.

There are also shortcut buttons that allow to perform all of these actions on all graphs in one step. The clean database restriction for the accept action still applies so some graphs may not be affected.

All of the actions can be invoked by administrator and the author of corresponding pipeline.
