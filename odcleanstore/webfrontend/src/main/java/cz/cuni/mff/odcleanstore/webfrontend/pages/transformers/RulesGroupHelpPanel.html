Group of rules

As multiple rules are often meant to be applied together the user can group them and then operate the group as a whole.

Each group is identified by its label and can (and should) come with a description of its semantical significance.

Notice: Each rule needs to belong to a group.

Content of rule groups may have uncommitted changes. Those changes are visible only to the author and need to be committed to be visible publicly. Do so with button.

Rule groups affect input graphs by being applied to them via their related transformer in one or more pipelines. By use of button it is possible to process those graphs again.

Groups can be debugged by using the . In that process user needs to specify input RDF triples to try applying the group of rules to it.

All groups can be modified (adding rules, removing rules) or deleted with corresponding buttons in the table.
