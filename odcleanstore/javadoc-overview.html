ODCleanStore is a server application for management of Linked Data written in Java. It stores data in RDF, processes them and provides integrated views on the data.

The application is divided into several modules. The codebase structure reflects that of modules and is described below.

Backend (odcleanstore/backend)
Contains predefined transformers, Query Execution and Conflict resolution modules implementations.
Comlib (odcleanstore/comlib)
Contains code related to sending data to Input Webservice, to be used on both the server and the client sides (e.g. Input Client and Input Webservice).
Core (odcleanstore/core)
Contains basic classes which are common to all modules. Most importantly interfaces for custom transformers.
Engine (odcleanstore/engine)
Contains the Engine component, which runs the web services and the data processing queue.
Input Client (odcleanstore/inputclient)
A library to be used in applications which serve as clients to ODCleanStore. It contains a simple SOAP WS client to the Input Webservice.
Installer (odcleanstore/installer)
A GUI installer of the whole application.
Simple Scraper (odcleanstore/simplescraper)
A trivial scraper application which can be used to upload data to ODCleanStore through a command line.
Simple Transformer (odcleanstore/simpletransformer)
An example custom transformer implementation.
Administration Frontend (odcleanstore/webfrontend)
Web application written in Java from which ODCleanStore can be managed. In Adiminstration Frontend, the user can define pipelines for data processing, rules for transformers, manage ontologies, Output Webservice settings etc.

For an in-depth description of individual classes and their interfaces, refer to this javadoc documentation. For a description of the architecture and design of the application and other general information, see the Programmer's Guide.
